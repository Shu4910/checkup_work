# Personal Productivity Tracker 仕様書

**バージョン**: 2.0
**更新日**: 2026-02-22
**対象環境**: Windows 10 / 11

---

## 1. 目的・背景

自分自身のPC使用状況を記録・可視化することで、日々の生産性を把握・改善する。
収集データはローカルPCにのみ保存し、外部への送信は一切行わない。

---

## 2. 対象ユーザー

個人ユーザー（自分自身のPCで使用する）

---

## 3. 機能要件

### 3.1 アクティビティ記録（メイン機能）

以下の情報をバックグラウンドで自動収集する。

| 収集項目 | 詳細 |
|----------|------|
| **タイムスタンプ** | 記録日時（YYYY-MM-DD HH:MM:SS） |
| **マシン名** | コンピュータ名 |
| **ユーザー名** | Windowsログインユーザー名 |
| **アクティブアプリ名** | フォアグラウンドのプロセス名（例: `chrome.exe`, `excel.exe`） |
| **ウィンドウタイトル** | アクティブウィンドウのタイトルバー文字列 |
| **ブラウザURL** | ブラウザがアクティブな場合、現在表示中のURL |
| **ページタイトル** | ブラウザがアクティブな場合、HTMLの `<title>` |
| **状態** | `active`（通常） / `locked`（ロック中・離席） |

#### 対応ブラウザ（URL・ページタイトル取得対象）

- Google Chrome
- Microsoft Edge
- Mozilla Firefox

### 3.2 記録タイミング

- **方式**: 定期ポーリング
- **間隔**: 30秒ごと（デフォルト、設定で変更可）
- **スリープ・ロック時の挙動**:

| 状態 | 挙動 |
|------|------|
| PC ロック中 | ポーリングは継続し、状態を `locked` として記録する |
| PC スリープ中 | Windowsのスリープイベントを検知してポーリングを**停止**する |
| PC スリープ復帰 | 復帰イベントを検知してポーリングを**自動再開**する |

### 3.3 ログの保存

- 保存形式: **CSV**
- 保存先: ローカルPC（`%USERPROFILE%\Documents\ProductivityTracker\logs\`）
  - `config.json` で変更可能
  - 環境変数（`%USERPROFILE%` 等）はコード側で展開する
- ファイル名: `activity_log_YYYY-MM-DD.csv`（1日1ファイル）
- 保存期間: **90日**（古いファイルは自動削除）

### 3.4 ログフォーマット

```csv
timestamp,machine_name,user_name,process_name,window_title,url,page_title,status
2026-02-22 09:01:00,MY-PC,username,chrome.exe,Google Chrome,https://mail.google.com/,受信トレイ - Gmail,active
2026-02-22 09:02:00,MY-PC,username,excel.exe,作業記録.xlsx - Microsoft Excel,,,active
2026-02-22 09:03:00,MY-PC,username,,,,,locked
```

### 3.5 本日のレポート機能

トレイメニューから「本日のレポートを表示」を選択すると、tkinterウィンドウで以下を表示する。

- 本日のアクティブ時間（合計）
- 本日の離席/ロック時間（合計）
- アプリ別使用時間（上位5件）

表示例:
```
本日のアクティビティ (2026-02-22)
────────────────────────────────
アクティブ:    5時間23分
離席/ロック:   1時間12分
────────────────────────────────
アプリ別使用時間 (上位5件):
  chrome.exe          2時間15分
  excel.exe           1時間30分
  code.exe              45分
```

### 3.6 常駐・自動起動

- PC起動時にタスクスケジューラで自動起動（オプション）
- **システムトレイにアイコンを表示**
- アイコンの右クリックメニュー:
  - 「現在の状態を表示」
  - 「本日のレポートを表示」
  - 「終了」（確認ダイアログあり）

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| **パフォーマンス** | CPU使用率 5%以下・メモリ 50MB以下を目標 |
| **バッテリー消費** | スリープ中はポーリング完全停止。ファイルI/Oは30秒に1回のみ |
| **対応OS** | Windows 10 (64bit) / Windows 11 |
| **ネットワーク** | 不要（完全オフライン動作） |
| **プライバシー** | データはローカルPCのみに保存。外部送信なし |

## 5. 技術スタック

| 用途 | 採用技術 |
|------|----------|
| 言語 | Python 3.12 |
| アクティブウィンドウ取得 | `pywin32`（Win32 API） |
| ブラウザURL取得 | `pywin32` / UIAutomation |
| スリープ検知 | `WM_POWERBROADCAST`（Win32 電源管理イベント） |
| ロック状態検知 | `WTS_SESSION_LOCK` イベント |
| データ保存 | CSV（標準ライブラリ `csv`） |
| システムトレイ | `pystray` + `Pillow` |
| ダイアログ | `tkinter`（スレッドセーフ） |
| パッケージング | PyInstaller（オプション） |

---

## 6. バッテリー消費への配慮

| 懸念点 | 対策 |
|--------|------|
| ポーリングによるCPU常時起動 | 30秒間隔 + スリープ時停止で影響を最小化 |
| ブラウザURL取得のコスト | UIAutomationはCPU負荷が一時的に上がる可能性あり。取得処理は1秒以内に完了するよう設計 |
| ファイルへの書き込み | 30秒に1レコードのみの軽量な書き込み |
| スリープ中の動作 | `WM_POWERBROADCAST` でスリープを検知し、ポーリングスレッドを即時停止 |

---

## 7. スコープ外（今回対象外）

- macOS / Linux 対応
- スクリーンショット機能
- Webダッシュボード・GUIビューア
- 複数PC間のログ自動収集・集約
- 特定サイト閲覧時のリアルタイムアラート
- 除外アプリ・URL設定

---

## 8. 決定事項サマリー

| 項目 | 決定内容 |
|------|----------|
| 記録タイミング | 定期ポーリング（30秒間隔） |
| ログ閲覧方法 | CSVファイル + トレイメニューからのレポート表示 |
| ログ管理場所 | ローカル保存のみ（`%USERPROFILE%\Documents\ProductivityTracker\logs`） |
| ロック中の扱い | `locked` として記録継続 |
| スリープ中の扱い | ポーリング停止（バッテリー配慮） |
| ログ保存期間 | 90日（自動ローテーション） |
| 除外設定 | なし |
| ダイアログ実装 | tkinter（win32api.MessageBox から移行、スレッドセーフ） |
